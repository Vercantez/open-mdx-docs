Cards are the primary way to present a grid of related destinations.

## Example

<CardGroup cols={2}>
  <Card title="Quickstart" href="/quickstart">
    Install, run the dev server, and add your first page.
  </Card>
  <Card title="Theming" href="/guides/theming">
    Brand colors, dark mode, and shadcn design tokens.
  </Card>
  <Card title="Search" href="/guides/search">
    Cloudflare AI Search and the local fallback index.
  </Card>
  <Card title="Deployment" href="/guides/deployment">
    Ship to Cloudflare Workers or Node.js.
  </Card>
</CardGroup>

## Usage

```mdx
<CardGroup cols={2}>
  <Card title="Quickstart" href="/quickstart">
    Get running in two minutes.
  </Card>
  <Card title="External" href="https://ui.shadcn.com">
    Cards also open external links in a new tab.
  </Card>
</CardGroup>
```

`cols` controls the preferred column count. On narrow screens the grid collapses automatically.

The default is two columns. Values of four or more use the three-column layout. When the final row has fewer cards than requested, those cards stretch to fill the row instead of leaving an empty cell.

## Horizontal cards

Use `horizontal` for a standalone call to action or banner.

<Card title="Open the quickstart" icon="rocket" href="/quickstart" horizontal>
  Install the package and start the local docs server.
</Card>

<Card title="Open the quickstart" icon="rocket" href="/quickstart" horizontal>
