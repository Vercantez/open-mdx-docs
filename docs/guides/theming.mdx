The site is themed with [shadcn/ui](https://ui.shadcn.com) design tokens on Tailwind CSS v4. Every color flows through CSS variables, so you can retheme the whole site without touching component code. Theme values are compiled into the site stylesheet at build time.

## Brand color

Set `colors` in `docs.json` — the same shape Mintlify uses:

```json
{
  "colors": {
    "primary": "oklch(0.55 0.2 250)",
    "light": "oklch(0.75 0.14 245)",
    "dark": "oklch(0.48 0.18 250)"
  }
}
```

- `primary` is used for light mode accents (links, active nav, callouts).
- `light` overrides the accent in dark mode; falls back to `primary`.
- These values compile to the `--docs-primary` CSS variables in the generated stylesheet.

Without a `colors` block or `theme.css`, the site uses a neutral zinc accent.

## Full shadcn theme

Create `theme.css` next to `docs.json` to control any shadcn token. You can paste a theme from [shadcn/ui themes](https://ui.shadcn.com/themes), [tweakcn](https://tweakcn.com), or your application's stylesheet.

```css
:root {
  --primary: oklch(0.55 0.2 250);
  --primary-foreground: oklch(0.985 0 0);
  --ring: oklch(0.55 0.2 250);
  --radius: 0.625rem;
}

.dark {
  --primary: oklch(0.75 0.14 245);
  --primary-foreground: oklch(0.141 0.005 285.823);
  --ring: oklch(0.75 0.14 245);
}
```

The cascade order is framework zinc defaults, `docs.json` `colors`, then `theme.css`. Because `theme.css` loads last, it can override the `--primary` mapping as well as background, border, radius, and other tokens. Changes to `theme.css` reload during local development.

## Fonts

Set proportional and monospace families with `fonts`. Google Fonts are loaded by default when either family is present:

```json
{
  "fonts": {
    "family": "Inter",
    "mono": "Geist Mono"
  }
}
```

Set `"source": "none"` when you load the fonts yourself. The families still populate the `--docs-font-sans` and `--docs-font-mono` CSS variables without adding Google Fonts links.

## Dark mode

Dark mode is class-based and driven by `next-themes`. The first visit follows the configured default, which is `system` unless you change it. The navbar button switches between light and dark, then saves that explicit choice in the browser.

Control behavior with `appearance` in `docs.json`:

```json
{
  "appearance": { "default": "dark", "strict": true }
}
```

- `default`: `light`, `dark`, or `system` (default).
- `strict`: when `true`, the theme toggle is hidden and visitors always get the default theme.

## shadcn tokens

The framework's default shadcn/ui tokens (`--background`, `--foreground`, `--card`, `--border`, `--primary`, ...) live in `app/app.css` under `:root` and `.dark`. Consumer sites override them in `theme.css`. Add framework components with the shadcn CLI:

```bash
bunx shadcn@latest add tooltip
```

The included `components.json` is preconfigured with the `~` alias and the `app/app.css` stylesheet.

## Code blocks

Syntax highlighting uses [Shiki](https://shiki.style) at build time with dual `github-light-default` / `github-dark-default` themes — zero JavaScript ships to the client for highlighting.
